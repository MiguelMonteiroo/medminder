# UI/UX Glossary

## Dose card
A card on the Hoje screen that represents one scheduled dose for today. It prioritizes quick actions such as Tomar, Adiar, Pular, and Detalhes.

## Medication card
A card that represents the medication entity itself. It belongs in medication management and detail contexts, not as the primary daily checklist item.

## Status badge
A compact visual label that communicates state without relying only on color. Current states include Pendente, Tomada, Pulada, Adiada, Perdida, Ativo, and Pausado.

## Primary action
The most important action for the user's current task. On the Hoje screen, the primary action is Tomar.

## Secondary action
An available but less prominent action, such as Adiar, Pular, Detalhes, or Desfazer.

## Destructive action
An action that removes or permanently changes data, such as Remover medicamento. Destructive actions require confirmation and should not appear as a routine dose action.

## Empty state
A helpful state shown when there is no content. It should explain what is missing and give the user a clear next step.
